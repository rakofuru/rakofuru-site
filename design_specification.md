# らこふる UI/UX刷新・設計仕様書

「ブルーベリー観光農園専門ポータル」として、ユーザーが「迷わず選べる」「行く理由が見つかる」体験を定義する。

## 1. デザインコンセプト: "Farm Real Estate"
不動産ポータル（SUUMOやHomes）のような「条件検索 → 比較 → 決定」のプロセスを適用する。
「なんとなく探す」ではなく「自分の条件（料金、時期、設備）に合う農園を特定する」体験を提供する。

---

## 2. トップページ UI構成 (比較・検索特化)

### 2-1. ヒーローエリア (The Filter Facade)
従来の「画像 + キャッチコピー」の上に、強力な**条件指定パネル**を配置する。
「エリア：千葉県」は固定（または目立たないトグル）とし、以下のフィルターを最優先する。

- **料金帯・プラン (Radio/Checkbox)**
  - [ ] 1000円以下
  - [ ] 1000〜2000円
  - [ ] 2000円以上
  - [ ] 食べ放題プランあり
- **設備・サービス (Checkbox)**
  - [ ] 駐車場あり
  - [ ] 雨天OK (屋根/ハウス)
  - [ ] 持ち帰り可
  - [ ] カフェ併設
- **営業時期 (Select/Radio)** ※"Now" をデフォルト
  - [ ] 今すぐ行ける (Current Month)
  - [ ] 6月
  - [ ] 7月
  - [ ] 8月

### 2-2. 検索結果リスト (Specs Card)
各カードは「物件スペック表」のように機能させる。画像を大きく見せつつ、データ密度を高める。

**カードレイアウト:**
- **上部**: メイン写真（角丸、現在の「営業中/シーズン外」バッジを重ねる）
- **中部 (リード)**:
  - 農園名 (Bold, Large)
  - 市区町村 (例: 千葉県 野田市) - アイコン付き
- **下部 (スペックグリッド)**:
  - 💰 **料金**: `1,500円` (大人1名)
  - 🚗 **駐車場**: `あり (15台)`
  - 📅 **期間**: `6月上旬〜8月`
  - 🏷️ **タグ**: `食べ放題` `持ち帰り可`

## 3. 農園詳細ページ UI構成 (決定特化)

ユーザーの思考フロー「場所は？ いくら？ やってる？ どう行く？」に完全同期させる。

### 3-1. ページ構成順序
1.  **Farm Hero (Basic Context)**
    -   背景：農園の高品質な写真（暗くしすぎない）
    -   前面：農園名、エリアバッジ、キャッチコピー
    -   アクション：「現在営業中」などのステータス表示
2.  **Unified Info Block (Decision Core)**
    -   **最も重要なセクション**。分散していた情報を1つの「概要カード」に統合。
    -   3カラム（スマホはスタック）:
        -   **Price**: 料金表（大人/子供/オプション）
        -   **Hours/Season**: 営業時間、開園期間、定休日
        -   **Access/Parking**: 駐車場台数、最寄りIC/駅からの距離
3.  **Gallery (Atmosphere)**
    -   農園の雰囲気がわかる写真グリッド（3-4枚）
4.  **Google Map (Location)**
    -   埋め込みマップ（全幅または大型表示）
5.  **Reviews (Proof)**
    -   口コミウィジェット（信頼性の担保）
6.  **Details (Deep Dive)**
    -   農園主からの挨拶、詳細なこだわり、栽培品種など（「もっと知りたい」人向け）
7.  **Sticky Footer (Mobile)**
    -   画面下部に「電話する」「公式サイト予約」を常駐させる（コンバージョン導線）

## 4. なぜこれが "じゃらん" より強いか

1.  **ノイズ・ゼロの専門性**
    -   じゃらんは「宿泊」「アクティビティ」が混在し、検索フィルターも汎用的すぎる。
    -   本サイトは「ブルーベリー狩りに必要な条件（品種、食べ放題、園内環境）」に特化したフィルターを提供できるため、**目的意識の高いユーザーにとっての到達速度が圧倒的に速い**。

2.  **「農園スペック」の標準化**
    -   一般的なポータルは、説明文の中に重要な条件（駐車場台数やトイレ有無）が埋もれがち。
    -   本サイトはこれらを「構造化データ」として抽出・標準化して表示するため、**タブを開いて比較検討する際のストレス・認知負荷が極めて低い**。

3.  **オーナーの顔が見える解像度**
    -   大手ポータルはシステム的な画一表示になりがち。
    -   本設計では、将来的にオーナー向け機能を追加することを前提に、**「詳細情報」セクションでオーナーの哲学やこだわりを語れるスペース**を確保している（今はbodyHtmlで表現）。これにより「スペックで選んで、最後は人で決める」体験を作れる。
